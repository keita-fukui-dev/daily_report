## わかったこと
- pullにはmerge型とrebase型があり、pullはなにか新しいcommitをするわけではないのでcommit分が残らないrebase型でpullをグローバル設定しておくと使い勝手が良い
- 直前のcommitをやり直すだけならgit commit --amendを使えばよく、それ以前の複数のcommitをやり直したいならgit rebase -i <コミットID>を使えばいい（iはinteractiveの略）
- commitの指定方法は”HEAD~値”か”HEAD^値”。キャレットの方はmergeした場合の親をしていする。
- commitは並べ替えたり、まとめたり、分割したりすることができる
- vimコマンド  yiw: カーソル上の単語をコピー  diw: カーソル上の単語を削除
- tagをつける理由はバグが発生したときにバージョンの切り戻しをしやすくしたり、いつ何をリリースしたかをわかりやすくするため（リリースポイントにタグ付けするのがおすすめ）
- tagには注釈付きタグと軽量版タグがある。注釈付きタグはオプション-aを付ける必要があり、タグ付けした人の情報、タグ付けした日時、注釈メッセージ、コミットが保存される。
- git stash（作業の一時避難）はbag fix時などに使うと便利
- ブランチの正確な意味は「コミットを指すポインタ（又はラベル）」
- HEADは現在作業中のコミットを指す特別なラベル。通常時はブランチにくっついて一緒に動く。
