## 取り組んだ課題一覧
- Git & GitHub    
## わかったこと
- HEADは自分が今いるブランチの最新のコミットを指す
- mainやfeatureブランチは今いる枝の最新のコミットを指し示すポインタ
- ブランチはコミットIDを記録したポインタ
- コンフリクト関連の事故が起きにくい運用ルール
    - 複数人で同じファイルを変更しない
    - pullやmergeする前に変更中の状態をなくしておく(commitやstashをしておく)
    - pullするときは、pullするブランチに移動してからpullする
    - コンフリクトしても慌てない
- mainブランチはリリース用にして、開発はトピックブランチを作成して進めるのが基本。そうすることで公開しているページにエラーが発生した際に簡単に前のバージョンに戻すことができる。
- rebaseでしてはいけないこと
    - GitHubにpushしたcommitをローカルでrebaseすること（リモートリポジトリにpushできなくなってしまう）
    - git push -f（絶対にNG）
- GitHub Flowの流れ
## 感じたこと
- リベースとマージの使い分け難しそう。とりあえずpushする前はリベースで履歴をできるだけきれいにしといて、push後はmergeにしとけばpushできなくなることはないので、そんな感じで行こう。
- 早くアウトプットしながら学びたい。気持ちは焦るけどコツコツ積み上げていこう。
