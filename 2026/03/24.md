## 取り組んだ課題一覧
- Git & GitHub
## わかったこと
- warp使えば現在のブランチが常に把握できる。VSCode上でも左下に現在のブランチが表示されていた。
- git initでディレクトリに作成される.gitディレクトリにGitで使用される機能がほぼ詰め込まれている。ローカルリポジトリに作成される圧縮ファイル、ツリーファイル、コミットファイルなどもここに作成される。
- 作業ディレクトリ、staging、リポジトリに分かれている。
- .gitの管理対象を外したいファイルがある場合は.gitignoreファイル内に記述しておく。環境変数を使うために.envファイルを使用する場合は、.envが外部に流出する恐れがあるので、.gitignoreファイルに入れる必要がある
- ブランチの作成と切り替えを同時にやりたい場合はgit switch -c ブランチ名と打てばいい
- Fast-forward mergeと3way mergeがあり、後者のほうがよく使われる。
- push, PR作成, マージ
