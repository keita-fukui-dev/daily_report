## わかったこと
- リダイレクト: ページ更新したときに直前の動作を重複して行ってしまうのを避けるために使用した。次にしてほしいリクエストを添えてクライアント側に返す
- ルーティング: URLに応じて処理を振り分ける仕組み。Node.js (Express)のルーティング記法を学んだ。    
- デバッグ手順: console.logも活用しつつ、Webページやコンソールなどのエラー表示確認や、エラー原因の仮説を立ててから取り掛かる
- HTTP: クライアントとサーバーが通信するときの手順やデータ形式を定めたルール
- ステートレス性: サーバーがクライアントの状態を保持しないという性質
- セッション管理: サーバー側がセッション情報とSIDをメモリーやデータベースに保存
- MySQLログイン: ターミナルから ">brew services start mysql@8.4" で起動させ "mysql --user=root --password" でパスワード入力してログイン
- 列の追加: ALTER TABLE テーブル名 ADD COLUMN カラム名 データ型;
- SPA: Single Page Application, Reactとかで作られているやつ。サクサク動くけど、SEO弱い
- SSR: Server Side Rendering, 遅いけどSEO強い。サーバサイドから表示させる。Node.js等で作る。SPAとSSRのいいとこ取りがNext.js
