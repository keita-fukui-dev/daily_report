## わかったこと

-- CloudFront --
- CloudFrontを使うときのポイントとして、cache TTLの設計（cache何秒保持するかは要件による）
- CloudWatch で監視（顧客からの問い合わせを待たずにエラーの発生件数などでチームで気付き対応していけるようにする）
- 100点の設計はない。利用者の使い方だったり、Webサービスの成長に応じてその100点が変わってくるから。だから定期的に分析して設計を最適にしていくという考え方が大事。
- 上記設計を見直すために、Popular Object Report で過去のリクエストやヒット、ミス（恐らくcacheが使えなかったこと）等を参考にすることができる。

-- WAF --
- WAFはWeb Application Firewallの略
- ハンズオンでは特定のIPからのみを拒否する設定を行ったが、実際に実務で必要となるWAFを1から設定するのは大変なので、あらかじめAWSで用意されているManaged Ruleから選択して設定することがよくある。

-- CLI --
- AWS Command Line Interfaceの略
- マネコンでできることはCLIでも可能。
- 大事なのは必要なコマンドを必要なときに調べることができるということ。調べ方を知っているということ。

-- SDK --
- AWS Software Development Kitの略
- アプリケーションに組み込むことができる言語ごとに用意されたビルド済みのコンポーネントとライブラリ
